Further Functions

  1. Functions are first-class objects.
  2. They can have properties and methods themselves.
  3. Functions have length property that returns the number of parameters the function has.

Call And Apply Methods

The call method can be use to set the value of "this" inside a function to an objecct that is
provided as the first argument.

Custom Properties

We can add properties to a function in the same way we add properties to any object in JavaScript.

Memoization

This provides result caching if a function takes some time to complete a return value, we can save
the return in a cach property.

Immediately Invoke Function Expressions - or IIFE

IIFE is an anonymous function that is invole as soon as it is define. This is clearly archieved by placing
parentheses at the end of the function. This function also has to be made into an expression which is done
by placing the whole declaration inside parentheses.

Function That Defines And Rewrite themselves

  1. Functions In JavaScript are able to not only call itself, but define itself, and even redefine itself.
  2. This is done by assigning an anonymous function to a variable that has the same name as the function.

Lossing Properties

If any property has previously been set on the function, this will be lost when the function redefines itself.

Recursive Functions.

A recursive function is one that invoke itself until a certain condition is met.

Callbacks

Callbacks are functions that passed to other functions as argument and then invoke inside the function they are
passed to.

AJAX

AJAX is a technique that allows web pages to communicate asynchronously with a server, and it dynamically update
web pages without reloading.
This enables data to be sent and recieved in the background

API's

An Application Programming Interface (API) is a collection of methods that allows extend access to another program or
service.

Hyper Text Transfer Protocol

HTTP is a standard or protocole use to transger data accross the web


The five commonly HTTP verbs are:
  1. GET: request to retrieve resources
  2. POST: request; usually use to create a resource, but can actually perform any task
  3. PUT: request to upset, which means to insert a resource or upadate it evenly
  4. PATCH: request to make partial updates to a resource
  5. DELETE: request to delete a resource
