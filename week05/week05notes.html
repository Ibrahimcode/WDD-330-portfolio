Errors, Exceptions, and Warning.

Errors are caursed when something goes wrong In a program.
The types of errors that commonly occurs are as follows:

  1. System Errors.
  2. Programmer Errors.
  3. User Errors

Exception is an error that produce a return value that can be used by
the program to deal with the error.

Stack Traces are sequence of functions or methods calls that lead to the
points where the error occured.

Warnings: A warning can occur if there is an error in the code that is not
enough to caurse the program to crash.

Strict Mode

Strict mode produce more exceptiona and warnings, and prohibits the use of
some depreciated features.

  1. Add the following string to the first line of a JavaScript file in other
    to use strict mode.
    • 'use strict'
  2. Strict mode can also be invoke on a per function basis.
  3. Modules are in a strict mode by default.

Linting Tools

Linting tools can be use to test the quality of a JavaScript code beyond the simply
using Strict mode.

Debugging

debugging is the process of finding out where bugs or errors occur in the code and
then dealing with them.

  1. The most basic form of debugging is to use the alert() method to alert a dialog
    message at certain points in the code.
  2. We can also use the console (console.log()) to log the valur of a variable at by
    different stages of the program during development.
